# 深い Module (Deep Modules)

『A Philosophy of Software Design』(John Ousterhout) より:

**深い Module** = 小さなインターフェース + 厚い実装

```
┌─────────────────────┐
│  小さなインターフェース │  ← メソッド少、引数シンプル
├─────────────────────┤
│                     │
│                     │
│    厚い実装          │  ← 複雑なロジックを隠蔽
│                     │
│                     │
└─────────────────────┘
```

**浅い Module** = 大きなインターフェース + 薄い実装（避ける）

```
┌─────────────────────────────────┐
│       大きなインターフェース        │  ← メソッド多、引数複雑
├─────────────────────────────────┤
│  薄い実装                         │  ← 素通しするだけ
└─────────────────────────────────┘
```

インターフェース設計時に問う:

- メソッド数を減らせないか？
- 引数をシンプルにできないか？
- もっと多くの複雑性を内部に隠せないか？
